Meteor Reproduction

Rspack client disconnects after a server-side edit

This is a minimal reproduction for a Meteor development bug. With rspack installed, changing a server file causes the browser client to disconnect and stop recovering after Meteor restarts the server. Without rspack, the same workflow keeps working.

Project context

  • The client is intentionally minimal.
  • The server is intentionally minimal.
  • The issue reproduces without app-specific client logic.
  • The issue disappears when rspack is not installed.

How to reproduce

  1. Start the app in development mode.
  2. Open the app in a browser and confirm the page loads.
  3. Edit a server-side file such as server/main.js.
  4. Save the file and wait for Meteor to restart the server.
  5. Watch the browser client lose connection and stop recovering.

Expected result

After a server-side save, the client should reconnect or reload and the app should remain usable.

Observed result

The browser stops loading the app after the server restart and shows the following Rspack client error: 

[Rspack Client Error] node:events:497
      throw er; // Unhandled 'error' event

Error: read ECONNRESET
